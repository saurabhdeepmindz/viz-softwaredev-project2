# Sprint v1 — PRD: Paper2Notebook

## Overview
A web application that allows researchers to upload any ML/AI research paper (PDF) and receive
a production-quality Jupyter notebook (.ipynb) that replicates the paper's core algorithm using
realistic synthetic data. The notebook is structured for professional use by researchers at
organizations like OpenAI, DeepMind, and Google Brain to accelerate paper replication workflows.

## Goals
- User can enter their OpenAI API key and upload a PDF in a beautiful web UI
- Backend parses the PDF and calls OpenAI `o3` to generate a structured, research-grade notebook
- Notebook is saved to `output/` and offered as a `.ipynb` download
- "Open in Colab" button lets users launch the notebook directly in Google Colab
- Animated processing screen keeps users engaged during generation (30–120s wait)

## User Stories
- As a researcher, I want to upload a PDF and get a replication notebook, so I can skip the boilerplate setup and focus on understanding the algorithm
- As a researcher, I want the notebook to use realistic synthetic data (not toy arrays), so I can immediately see how the method performs at scale
- As a researcher, I want to open the notebook directly in Colab, so I don't have to install anything locally
- As a user, I want to see progress messages while waiting, so I know the system is working
- As a user, I want a beautiful, focused UI inspired by claude.ai, so the tool feels professional

## Technical Architecture

### Stack
- **Frontend**: Next.js 14 (App Router) + Tailwind CSS + shadcn/ui
- **Font/Theme**: Geist font (same as claude.ai) — dark/light minimal, purple accent, clean card layouts
- **Backend**: Next.js API routes (streaming via SSE for progress updates)
- **PDF Parsing**: `pdf-parse` (Node.js) — extracts full text from uploaded PDF
- **LLM**: OpenAI `o3` via `openai` Node SDK — reasoning model for deep paper comprehension
- **Notebook Output**: `@jupyterlab/nbformat` compatible JSON, written to `output/<slug>.ipynb`
- **Colab Link**: Base64-encode notebook → GitHub Gist API → generate Colab URL

### Component Diagram
```
Browser
  │
  ├── / (Landing)
  │     └── ApiKeyInput + FileUpload → POST /api/generate
  │
  ├── /processing (SSE stream)
  │     └── ProgressFeed (animated steps) ← GET /api/status/:jobId
  │
  └── /result
        ├── DownloadButton (.ipynb)
        └── OpenInColabButton (gist URL)

Server (Next.js API Routes)
  │
  ├── POST /api/generate
  │     ├── parse PDF (pdf-parse)
  │     ├── extract: title, abstract, algorithm, equations, datasets
  │     ├── call o3 with structured system prompt
  │     └── write output/<slug>.ipynb
  │
  ├── GET /api/download/:slug  →  serve .ipynb file
  │
  └── POST /api/colab/:slug    →  create GitHub Gist → return Colab URL
```

### Data Flow
```
PDF Upload → pdf-parse → raw text
         → o3 (system prompt: paper analyst + notebook architect)
         → structured JSON (cells array with metadata)
         → nbformat v4 assembly
         → output/<slug>.ipynb
         → download / Colab gist
```

### Notebook Structure (generated by o3)
Each generated notebook must include these sections as top-level markdown headers:
1. **Paper Summary** — title, authors, venue, one-para abstract
2. **Core Contributions** — bulleted breakdown of novelty
3. **Mathematical Foundations** — all key equations rendered in LaTeX
4. **Algorithm Walkthrough** — pseudocode + line-by-line explanation
5. **Synthetic Data Generation** — realistic data (e.g., for vision: 10k ImageNet-like embeddings; for NLP: 50k token sequences with proper vocab distributions)
6. **Implementation** — full Python implementation of the core method
7. **Experiments & Ablations** — reproduce key results table with synthetic data
8. **Visualization** — plots matching figures in the paper
9. **Discussion & Limitations** — what the paper doesn't address
10. **References** — formatted citations

### o3 System Prompt Strategy
- Role: "You are a world-class ML research engineer at a top AI lab..."
- Output format: strict JSON array of notebook cells
- Instruction: use realistic synthetic data (proper statistical distributions, correct dimensionality)
- Instruction: all code must be runnable end-to-end in Colab (no missing imports)
- Instruction: include inline comments explaining *why*, not just *what*

## Out of Scope (v2+)
- User authentication / accounts
- Usage tracking / rate limiting
- Multiple simultaneous jobs per user
- Persistent job history
- Custom notebook templates
- Support for multi-paper inputs
- Fine-tuned models

## Dependencies
- None (greenfield project)
- OpenAI API key provided by user at runtime (not stored)
- Optional: GitHub token for Gist creation (Colab feature)
